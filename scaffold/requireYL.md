# require Source Code Analysis

## Use Cases for require
### Module Loading Types:
* Load built-in modules: `require('fs')`
* Load node_modules modules: `require('ejs')`
* Load local modules: `require('./utils')`
### Supported File Types:
* Load .js files
* Load .json files
* Load .node files
* Load .mjs files
* Load other file types

## The Module Object
* id: Source file path, e.g., `/Users/sam/Desktop/vue-test/imooc-test/bin/ejs/index.js`
* path: The folder containing the source file, generated by `path.dirname(id)`
* exports: Module output content, defaults to `{}`
* parent: Parent module information
* filename: Source file path
* loaded: Whether loading is complete
* children: Collection of child module objects
* paths: Module lookup scope

## Summary of require Execution Flow
1. `relativeResolveCache[relResolveCacheIdentifier]` Look up cached path
2. `Module._cache[filename]` Look up cached module
3. `Module._resolveFilename` Look up the real path of the module
4. `loadNativeModule` Load built-in module
5. `new Module` Instantiate Module object
6. `module.load(filename)` Load module
7. `findLongestRegisteredExtension` Get file extension
8. `Module._extensions[extension](this, filename)` Parse and execute the module
9. `module._compile` Compile module code
10. `compileFunction` Generate executable function from module code
11. `exports, require, module, filename, dirname` Generate input parameters
12. `compiledWrapper.call` Execute the module function
13. `return module.exports` Output module result
